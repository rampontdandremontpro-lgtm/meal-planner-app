# ADR-009 — Utilisation de Storybook

## Statut
Accepté

## Contexte
Le frontend contient plusieurs composants UI et pages qui doivent être testés visuellement.

## Décision
Nous avons choisi d’utiliser Storybook pour documenter et visualiser certains composants du frontend.

## Conséquences positives
- Test visuel des composants
- Documentation UI
- Développement plus rapide
- Possibilité de vérifier les états d’un composant sans lancer tout le parcours utilisateur

## Conséquences négatives
- Configuration supplémentaire
- Maintenance des stories nécessaire

## Alternatives envisagées
- Tester uniquement dans l’application
- Utiliser uniquement des captures d’écran
