# ADR-009 — Utilisation de Storybook

## Statut
Accepté

## Contexte
Le projet nécessite de développer et tester les composants UI indépendamment.

## Décision
Utilisation de Storybook pour le frontend.

## Conséquences positives
- Test des composants isolés
- Documentation visuelle
- Développement plus rapide

## Conséquences négatives
- Configuration initiale
- Maintenance supplémentaire

## Alternatives envisagées
- Tests directement dans l’application
