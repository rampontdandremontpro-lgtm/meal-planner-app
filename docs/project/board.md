# Board du projet

Ce document présente l’organisation et le suivi des tâches du projet **Meal Planner**.

---

## Répartition par jours

### Jour 1 — Base + Authentification

Objectifs :

- initialiser les projets
- mettre en place la structure
- créer l’authentification
- tester la connexion

### Jour 2 — Recettes

Objectifs :

- afficher les recettes
- gérer les recettes externes
- créer les recettes locales
- relier le frontend, le mobile et le backend

### Jour 3 — Planning + Liste de courses

Objectifs :

- créer le planning semaine
- ajouter une recette au planning
- générer la liste de courses
- gérer les ingrédients automatiques et manuels

### Jour 4 — Corrections + UX

Objectifs :

- corriger les bugs
- améliorer l’expérience utilisateur
- stabiliser l’intégration backend / frontend / mobile

### Jour 5 — Documentation + Présentation

Objectifs :

- ajouter Swagger
- rédiger les ADR
- finaliser le README
- préparer les diagrammes
- préparer la présentation orale
